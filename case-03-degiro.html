Case 03 Redesign · Trading Platform

DEGIRO Trading View

A concept redesign of DEGIRO's portfolio dashboard, addressing real user friction through clearer hierarchy, a modern visual identity, and actions placed next to the data that justifies them.

Concept redesign

DEGIRO's interface hasn't kept pace with modern trading platforms. Strong functionality is let down by flat hierarchy, cluttered layouts, and a visual language that makes fast scanning harder than it should be.

Background

A platform with real friction

DEGIRO is one of Europe's largest retail brokers, serving over 3 million customers across the continent, including a significant user base in Sweden. It built its reputation on extremely low trading fees, making markets accessible to a wider audience than traditional brokers allowed.

But the interface has not kept pace with the product's growth. User reviews consistently describe the platform as "clunky" and note that it "was fine in 2018, but it's not good enough" for today's standards. The functionality is all there. The experience of using it is where the gap exists.

This redesign concept focuses on DEGIRO's portfolio and trading view, the screen active traders return to most frequently. The goal is not to change what the platform does, but to make it significantly clearer and faster to use.

Analysis

What the current interface gets wrong

The current DEGIRO desktop interface uses a light, flat visual language across the entire product. Every element, from navigation tabs to portfolio data to order inputs, receives roughly equal visual weight. There is no clear hierarchy directing the user's eye toward what matters most.

For a user who checks their portfolio briefly, this is manageable. For an active trader making time-sensitive decisions, it creates real friction: too much scanning, too little structure.

A.01

Visual hierarchy audit

Mapped the current portfolio view against the tasks active traders perform most: checking position performance, reading price changes, and placing orders. Identified where visual weight matched task importance, and where it didn't.

A.02

User feedback review

Reviewed public user feedback from app stores and trading forums. Consistent themes included cluttered navigation, poor readability of price data, and the disconnect between viewing a position and acting on it.

A.03

Competitive reference

Reviewed interfaces from Trading 212, Saxo Bank, and Interactive Brokers to understand how modern trading platforms handle information density, colour coding, and the relationship between data and action.

Findings

What the analysis revealed

  • All elements compete equally for attention. Price data, navigation, account info, and order inputs sit at the same visual level, making it hard to know where to look first.
  • Positive and negative price changes rely solely on colour with no structural differentiation. On a bright white background, the contrast is lower than it needs to be for fast scanning.
  • The order placement flow is visually disconnected from the portfolio data. Users must shift their focus significantly between seeing a price and acting on it.
  • The interface feels dated because it is. The flat, form-heavy aesthetic belongs to an earlier era of web design and reads as low-confidence compared to modern fintech products.
Key insight

DEGIRO's core product is strong. The redesign challenge is not about adding features, it's about making the existing functionality feel as modern and confident as the platform's actual capabilities.

Design Goals

What the redesign needed to achieve

G.01

Establish a clear visual hierarchy

Portfolio performance and price changes should dominate. Navigation and secondary controls should recede. The trader's eye should land in the right place without effort.

G.02

Modernise the visual identity without abandoning the brand

DEGIRO's orange accent is recognisable and should survive the redesign. What changes is the background, the typography weight, and the density of visual elements around it.

G.03

Shorten the path from information to action

The selected position and the order panel should live in the same view. A trader should be able to see a price and place an order without moving their eyes across the screen.

G.04

Improve scannability of price data

Green and red changes read faster on dark backgrounds. Monospace numerals prevent column shift as values update. Both are small changes with significant impact on usability.

Key Changes

What changed and why

01

Dark background

Reduces eye strain during extended sessions and dramatically improves contrast for colour-coded price data. Green gains and red losses are immediately readable without squinting.

02

Monospace price numerals

Using Space Grotesk in tabular figures prevents column shift as prices update in real time. A small detail with a significant impact on readability during live trading.

03

Selected row highlight

A subtle cyan tint shows which position is active. The side panel updates to reflect it, showing the price, sparkline, and order inputs for that specific position only.

04

Order panel in the sidebar

Buy, sell, and order inputs now live directly beside the selected position's data. The user's eye doesn't need to travel. Information and action are in the same view.

05

Sparkline for selected position

A 7-day sparkline gives directional context for the selected stock without opening a full chart. Enough to inform a quick decision. More detail is one click away.

06

DEGIRO orange retained

The brand's signature orange stays in the logo. The redesign evolves the visual identity without replacing it, making the result feel like a natural progression rather than a replacement.

Reflection

What this exercise showed me

The most interesting constraint in this project was working within an existing brand. DEGIRO's orange is recognisable and carries real equity with its users. The challenge was to modernise the visual language around it without touching the one element most users associate with the product.

That constraint pushed the redesign in a better direction. Rather than a clean-slate dark theme, the result is a dark interface with a warm orange anchor, which actually reads as more distinctive than either a pure dark or pure light design would.

The sidebar order panel. Collapsing the distance between position data and order placement made the flow feel cohesive in a way that was immediately visible in the before/after comparison.

Whether DEGIRO's existing users, many of whom are accustomed to the light interface, would adapt quickly to the dark design or experience initial friction. That's a real usability question the concept doesn't yet answer.

Visual identity is not decoration. In a trading context, how data looks directly affects how fast it's processed and how confident the resulting decision feels. Clarity is a functional requirement, not an aesthetic preference.

The job involves redesigning existing dashboards to harmonise with a new visual identity. This is a direct demonstration of that: a real product, a clear problem, a considered redesign that respects the original while making it significantly better.
